# 📚 Resources

Reference material organised by **topic**, not by urgency: source notes on books/articles/videos, how-to guides, checklists, anything you might want again someday but that serves no current project or area.

## How to use it

- Every book, article, video or podcast worth keeping gets a note from [[07 - Templates/Source|the Source template]] — bibliographic info, *your* summary, and the quotes that hit.
- Group with subfolders per topic only once a topic has ~5+ notes. Premature folders are clutter; links and tags do most of the work.
- A resource that becomes relevant to a project? **Link it** from the project note — don't move it. Notes can serve many masters via links.

## The distillation rule

Resources are *other people's ideas*. When one changes how **you** think, write that realisation as your own atomic note in [[05 - Zettelkasten/Zettelkasten - How to write permanent notes|the Zettelkasten]] and link back to the source. Highlights you never process are just hoarding with extra steps.
