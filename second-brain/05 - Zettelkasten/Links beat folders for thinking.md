# Links beat folders for thinking

A folder puts a note in exactly one place, decided at filing time — before you know what the note will be useful for. A link connects a note to any number of contexts, added whenever a connection is noticed.

Folders answer "where do I put this?"; links answer "what does this mean next to that?" — and the second question is the one that produces new ideas. That's why this vault keeps folders shallow and boring (PARA is just a filing convention for *retrieval*) while the real structure lives in the link graph.

Practical consequence: time spent perfecting folder taxonomy is better spent writing one more link.
