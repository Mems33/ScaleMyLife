# ⚙️ How this vault works

Three proven methods, glued together: **CODE** for the flow of information, **PARA** for filing, **Zettelkasten** for thinking. You don't need to study any of them — the vault embodies them.

## CODE — the flow

1. **Capture** — anything that resonates → [[00 - Inbox/Inbox|Inbox]], zero friction, no organising.
2. **Organise** — in reviews, file by *actionability* (PARA below), not by topic.
3. **Distil** — progressively summarise: bold the best lines of a source note; when an idea is truly yours, write it as a [[05 - Zettelkasten/Zettelkasten - How to write permanent notes|permanent note]].
4. **Express** — use the notes: write, build, decide, teach. A second brain that only ingests is a landfill.

## PARA — the filing question

File everything by asking **"where will I use this?"** — never "what topic is this?"

- **P**rojects — active, has a goal and an end → `02 - Projects`
- **A**reas — ongoing responsibility, has a standard → `03 - Areas`
- **R**esources — might be useful someday → `04 - Resources`
- **A**rchive — none of the above anymore → `06 - Archive`

## Zettelkasten — the thinking layer

Atomic, self-written, densely linked permanent notes in `05 - Zettelkasten`. This is the part that compounds: the graph gets smarter every week, and old notes start answering new questions.

## Conventions

- **Frontmatter** on every note: `type` (daily / project / area / source / person / zettel / meta), plus `status`, `area`, `deadline` where relevant. Dataview dashboards key off these.
- **Titles are sentences** for zettels, `Project - X` for projects, plain names elsewhere.
- **Link generously**, folder sparingly — see [[Links beat folders for thinking]].
- **Never delete, archive** — storage is free, regret isn't.
- Attachments land in `99 - Attachments` automatically.

## The one rule that outranks the rest

**The system serves the work, not the other way around.** If a convention here creates friction, change the convention. A messy vault you use daily beats a perfect one you're afraid to touch.
